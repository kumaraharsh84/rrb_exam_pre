# Supabase Live Leaderboard Setup Guide

This guide describes how to set up your Supabase database to enable the live, shared leaderboard for your RRB Exam Prep application.

## 1. Create a Supabase Project

1. Go to [supabase.com](https://supabase.com) and log in or sign up.
2. Click **New Project** and select/create an organization.
3. Choose a project name, database password, and region, then click **Create new project**.

## 2. Retrieve API Credentials

1. Once the project is created, navigate to **Project Settings** (cog icon on the left panel) > **API**.
2. Under **Project API keys**, copy the **Project URL** and the **anon (public)** key.
3. Open [frontend/js/config.js](file:///c:/Users/kumar/OneDrive/Desktop/project/rrb%20exam%20pre/rrb-exam-prep/frontend/js/config.js) in your editor and replace the placeholder values:
   ```javascript
   const SUPABASE_URL = "https://your-project-id.supabase.co"; // Replace with Project URL
   const SUPABASE_KEY = "your-anon-key";                      // Replace with anon public key
   ```

## 3. Create the Database Table & Security Policies

1. Go to the **SQL Editor** in the left sidebar menu in your Supabase Dashboard.
2. Click **New query** and paste the following SQL script:

```sql
-- Create the leaderboard table
create table public.leaderboard (
  id bigint generated by default as identity primary key,
  username text not null,
  exam text not null,
  points integer not null default 0,
  tests_done integer not null default 0,
  best_score integer not null default 0,
  updated_at timestamp with time zone default timezone('utc'::text, now()) not null,
  constraint unique_username_exam unique (username, exam)
);

-- Enable Row Level Security (RLS)
alter table public.leaderboard enable row level security;

-- Create Policies to allow anonymous public access (Select, Insert, Update/Upsert)
create policy "Allow public read access"
  on public.leaderboard for select
  using (true);

create policy "Allow public insert access"
  on public.leaderboard for insert
  with check (true);

create policy "Allow public update access"
  on public.leaderboard for update
  using (true)
  with check (true);
```

3. Click **Run** to execute the script. The `leaderboard` table is now configured and ready to sync live scores across all users!
